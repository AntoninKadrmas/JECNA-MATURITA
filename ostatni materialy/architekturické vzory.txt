Používá se v všech programovacích jazycích
MVC
    model, view, controller
    model = práce s informacema
    view se může zeptat přímo modelu 
    view může se ptát a číst z modelu ale zapisovat a měnit může jenom controller
P2P
    každý pír se chová vlastně jak jako server tak i klient, 
    je na tom založený torent
    může jeden vypadnout, a síť stále může fungovat
Client Server
    thin = klient využívá server celá plakace a práce s daty zůstavají na satraně serveru. po vypnutí se vše smaže a pod (hadej číslo)
        vždy závislý na serveru vše jede na serveru
    thick = využívá cpu a ukládá data klient a servert si navzájem posílá data, klientí data jsou využívána
        bere si data z počítače a uložené data nějaké informace dostává informace z serveru
    hybrid = nepužívá se cpu a posílá se data mezi klienty neukládají se (např skype)
        nebere data z počítače ale přenáší je 
    když zkončí server tak už klinti mají smůlu

    spojový seznam komunikujeme po řadě
