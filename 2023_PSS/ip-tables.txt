probíhají pravidla chronologicky

INPUT = kontrola na vstupu
OUTPUT = kontrola na výstupu
FORWARD = odkud kam ma dve síťovky s masquarade(překlad addressy) jako nat
    ACCEPT = v pořádku propustí
    DROP = zahodí a nedává o sobě dále vědět
    REJECT = zamítne a pošle info odesílateli
    == pokud se vyhodnotí jedna z předchozích tak už se dále v chainu pravidla nevyhodnocují
    == pokud dojde až nakonec použije se defaultní politika (default ACCEPT)
socket = kombinace ip addresy a portu

iptables -P INPUT DROP
